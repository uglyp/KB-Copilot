# 智能知识问答 Agent（Copilot）产品需求文档（PRD）

| 属性 | 内容 |
|------|------|
| **文档版本** | v1.5 |
| **产品代号** | KB-Copilot |
| **更新日期** | 2026-03-23 |
| **产品定位** | 以 **多轮智能问答** 为核心体验，集成 **知识库上传、解析、索引与检索（RAG）**，回答需 **可引用、可追溯**；形态上对标 **「对话式知识助手 + 私有知识库」**（能力参考：ima.copilot 一类产品，非实现其代码或品牌） |

### 推荐技术栈（与实现方对齐）

| 层级 | 选型 |
|------|------|
| **前端** | **Vue 3 + TypeScript + Vite**；**Element Plus**（管理端 + 对话布局） |
| **后端** | **Python 3.11+**，**FastAPI**；**SQLAlchemy 2** + **Alembic** |
| **主数据库** | **MySQL 8.0+**（会话、用户、知识元数据、审计） |
| **缓存/队列** | **Redis**（会话上下文缓存、限流、异步解析/向量化任务） |
| **向量检索** | **Qdrant**（Docker 单节点）或 **应用层小规模向量**（仅原型）；与 MySQL 元数据关联 |
| **大模型** | **可配置多提供方**（OpenAI 兼容为主，见 §5.4）；**用户端「设置」即可配置**，不限于管理端 |

## 1. 背景与目标

### 1.1 背景

用户希望在 **自然语言对话** 中完成「查内部资料、对材料提问、要结论与依据」，而不是在多个文件夹里翻找。产品需提供：**对话式 Agent** + **可维护的知识库**（上传、更新、按库问答），并降低幻觉带来的信任风险——通过 **引用片段** 让用户自行核对。

### 1.2 产品目标

| 目标 | 说明 | 指标方向（示例） |
|------|------|------------------|
| **问得清** | 多轮澄清、上下文连贯 | 会话轮次、任务完成率 |
| **答有据** | 答案关键结论能对应 **知识库片段引用** | 有引用轮次占比、用户点踩率 |
| **库好用** | 上传后自动解析、可重建索引、状态可见 | 索引成功率、从上传到可问耗时 |
| **可管控** | 权限、审计、预算（Token/调用） | 越权为 0、成本可观测 |

### 1.3 非目标（MVP / 第一期）

- 不承诺 **完全替代专业搜索/BI**；复杂数值分析以 **引用原文 + 引导用户自行计算** 为主。
- **不内置** 完整工单/IM；可在 P2 通过 **插件/Webhook** 外接。
- 不在首版承诺 **多模态深度理解**（复杂图表、视频内容理解等），以 **文本与常规文档** 为主。

## 2. 用户与角色

| 角色 | 典型诉求 |
|------|----------|
| **终端用户** | **完成模型设置后** 进入聊天；可选 **针对哪个知识库** 提问；需要 **流式输出** 与 **引用来源**；在 **「设置 → 模型与密钥」** 中 **自行添加提供方、填写 Key、选择默认对话/嵌入模型**（密钥仅服务端存储） |
| **知识管理员** | 创建知识库、上传/删除文档、查看 **解析与索引状态**、触发 **重建索引** |
| **系统管理员** | （可选）**全系统默认模型**、预算与限流、租户级策略、审计导出；**不独占**模型配置能力——**普通用户即可在端上完成自有厂商接入** |

> MVP 可合并管理员角色为 1～2 人；**模型配置以用户端为主路径**，管理端为 **可选增强**（如企业统一下发 Key 时的策略）。

## 3. 核心业务概念

| 概念 | 说明 |
|------|------|
| **会话（Conversation）** | 用户与 Agent 的多轮对话单元 |
| **消息（Message）** | user / assistant / system；assistant 可携带 **citations[]** |
| **知识库（Knowledge Base）** | 一类资料的边界；对话时可 **单选/多选（P1）** 限定检索范围 |
| **文档（Source Document）** | 上传的文件或粘贴的文本；经解析 → 切块 → 向量化 |
| **Chunk** | 检索与引用的最小片段；回答中展示 **标题/页码/片段摘要** |
| **RAG** | 检索相关 Chunk → 作为上下文 → LLM 生成；**引用必须指向 Chunk** |
| **模型提供方（Provider）** | 某一厂商或自建网关的接入配置，例如 **API Base URL、鉴权方式、协议类型**；可挂在 **用户（私有）** 或 **系统（可选默认）** |
| **模型实例（Model Profile）** | 在提供方下的一条可用模型，区分用途：**对话（chat）**、**文本嵌入（embedding）** 等；可设 **默认**、**是否启用** |
| **模型就绪（Model Ready）** | 当前用户已在库中配置 **已启用的默认 chat 模型** 与 **默认 embedding 模型**（各至少一条）；**未就绪时不得使用对话与依赖嵌入的能力**（见 A6c） |
| **解析优先级（就绪后）** | 会话调用时：**用户已保存的提供方与默认模型**；**环境变量** 仅用于 **开发联调** 或 **表单预填**，**不作为生产环境免设置即可聊天的替代路径**（见 §5.4） |

## 4. 业务场景与优先级

### 4.1 P0（MVP）— Agent + 知识库最小闭环

| ID | 场景 | 说明 |
|----|------|------|
| A1 | **多轮对话** | 上下文窗口可配置；支持 **流式输出（SSE）** |
| A2 | **选择知识库** | 新建会话时绑定 1 个 KB（MVP）；无库可走 **通用提示** 或禁止（可配置） |
| A3 | **RAG 问答** | 先检索 Top-K Chunk → 再生成；**回复中展示引用列表**（文档名、片段、可选定位） |
| A4 | **知识库管理** | 创建 KB、上传 **PDF/TXT/Markdown**（MVP 格式可收敛）、列表与删除 |
| A5 | **解析与索引** | 异步：解析文本 → 切块 → embedding → 写入向量库；**状态**：排队/处理中/就绪/失败 |
| A6 | **鉴权与审计** | 登录；记录 **谁问了什么、用了哪个 KB**（摘要可配置脱敏） |
| A6b | **用户端「模型设置」页（必选）** | MVP **必须上线**：登录用户在 **「设置 → 模型与密钥」** 中完成 **至少一个提供方** 的配置（Base URL、API Key、默认 **对话** / **嵌入** 模型 ID 等）；协议 **OpenAI 兼容**；密钥 **仅服务端、加密存储** |
| A6c | **未配置禁止进入聊天** | 当用户 **模型未就绪** 时：**禁止进入 `/chat` 聊天页**（路由拦截，重定向至 **`/settings/models`**）；可展示 **ElMessage** 类提示「请先完成模型配置」；**创建会话、发送消息** 等接口在服务端 **同步校验**，返回统一错误码（如 `MODEL_NOT_READY`） |

### 4.2 P1 — 体验与运营

| ID | 场景 |
|----|------|
| A7 | **混合检索**：关键词（MySQL）+ 向量融合，降低漏检 |
| A8 | **反馈**：点踩/点赞；可选填原因 |
| A9 | **多知识库同时检索**（若权限允许） |
| A10 | **对话历史**：按会话列表查看、删除 |
| A14 | **多提供方与扩展协议（P1）** | 在用户端设置中 **添加多个提供方**、扩展 **Azure/国产云** 等（`provider_type`）；**组织级管理端** 仍为 P2 |
| A15 | **多对话模型与选模（P1）** | 同一提供方下 **多个对话模型**；**会话内下拉选模**（列表来自用户已启用模型）；MVP 仅需 **单默认对话模型 + 单默认嵌入模型** 即可 |
| A16 | **嵌入模型独立配置** | 与对话模型解耦；**更换嵌入模型** 需提示 **重建知识库向量索引** |

### 4.3 P2

| ID | 场景 |
|----|------|
| A11 | **联网搜索**（需合规与来源展示） |
| A12 | **工具调用**（查外部 API、执行脚本等，强风控） |
| A13 | **多租户/组织**、SSO |

## 5. 功能清单

### 5.1 对话与 Agent（核心）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 新建会话、发送消息 | 用户输入 → SSE 流式返回 | P0 |
| 系统提示词模板 | 约束「无检索结果则拒答或说明」、输出引用格式 | P0 |
| 引用渲染 | 侧栏或消息下方 **可展开引用卡片** | P0 |
| 停止生成 | 前端中止 SSE | P0 |
| 导航入口 | 已登录用户可从 **顶栏/侧栏** 进入 **「设置 → 模型与密钥」**（路由如 `/settings/models`） | P0 |
| **聊天页门禁** | **Vue Router `beforeEach`**（或全局路由守卫）：若 **`GET /v1/me/model-readiness`（或等价）** 返回 `ready=false`，则 **禁止进入 `/chat`**，**重定向到 `/settings/models`**（设置页本身、登录页除外）；登录成功后的 **默认跳转**：未就绪 → 设置页，已就绪 → 聊天或原 intended 路由 | P0 |
| 会话标题自动生成（可选） | 首条消息摘要为标题 | P1 |

### 5.2 知识库与内容

| 功能 | 说明 | 优先级 |
|------|------|--------|
| KB CRUD | 名称、描述 | P0 |
| 上传文档 | 大小与类型白名单；进度条 | P0 |
| 解析队列 | Redis + Worker；失败可重试、查看错误信息 | P0 |
| 文档列表与删除 | 删除后触发 **索引失效** | P0 |
| 索引依赖嵌入 | **模型未就绪** 时：**禁止触发/排队** 需 embedding 的解析任务（上传按钮禁用或提示先去 **模型设置**） | P0 |
| 富文本手动条目（可选） | 非文件，直接录入 Markdown | P1 |

### 5.3 检索与生成（后端）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| Embedding | 与向量库维度一致 | P0 |
| 向量检索 | Top-K，过滤 `kb_id` | P0 |
| Prompt 组装 | system + retrieved chunks + user message | P0 |
| 无结果策略 | 明确告知「库中无相关内容」 | P0 |
| 调用链选模 | 生成与嵌入均从 **当前生效的 Model Profile** 解析 endpoint 与模型名 | P0 |

### 5.4 模型提供方与模型配置（通用能力）

> 目标：不绑定单一厂商；**终端用户在「设置/模型」中即可完成接入**，无需依赖 **单独的管理后台**（管理端仅为可选能力）。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **用户端配置入口** | 路由 **「设置 → 模型与密钥」**（或 `/settings/models`）：**MVP 必须交付**；列表、添加、编辑、删除 **自己的** 提供方与模型 | **P0** |
| **提供方（Provider）** | 名称、**API Base URL**、**API Key**（仅存服务端，列表 **脱敏**）、协议类型（默认 **OpenAI Compatible**）、可选 **额外 Header**；**`user_id` 归属** | **P0** 用户端 CRUD |
| **模型实例（Model）** | 展示名、`model_id`、用途 **chat / embedding**、是否启用、**用户级默认**（同用途唯一默认）；MVP **至少各设一个默认 chat 与一个默认 embedding** | **P0** |
| **健康检查 / 连通性测试** | 用户点击 **「测试连接」** 验证 chat / embedding 各一次（或合并为一次探活） | **P0** |
| **会话选模** | 聊天区 **下拉切换对话模型**（相对默认模型的增量能力） | **P1** |
| **密钥安全** | Key **加密存储**（AES 等）；审计 **用户本人** 的配置变更 | **P0** |
| **开发环境绕过（可选）** | 仅 **开发/测试**：可通过环境变量 **`ALLOW_CHAT_WITHOUT_MODEL_SETUP=true`** 跳过路由门禁（**禁止用于生产**） | 开发体验 |
| **扩展点** | 预留 `provider_type`（如 `openai_compatible`、`azure_openai`、`custom`） | 设计期 |
| **管理端（可选）** | 若企业需要 **统一下发策略**，可增加 **只读审计** 或 **组织级 Provider**（`tenant_id`），**不替代**用户自带 Key 的场景 | P2 |

**MVP 落地说明（强制）**：  
- **用户端「模型设置」页为上线门禁**：未交付则 **不视为 MVP 完成**。  
- **未配置模型禁止进入聊天页（A6c）**：与 **模型就绪** 定义一致；**生产环境** 不得以「仅配置环境变量、不落库」的方式绕过。  
- **P1**：多提供方并存、**会话内选模**（A14/A15）在 MVP 单默认模型跑通后再迭代。

### 5.5 权限与安全

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 用户只能访问被授权的 KB | RBAC 或用户-KB 关联表 | P0 |
| API Key / 模型 Key 仅服务端 | **加密落库**；见 §5.4 | P0 |
| **聊天与模型 API 一致性** | `POST /v1/conversations`、`POST .../messages` 等：**模型未就绪** 时返回 **412** 或 **403**，body 含 **`code: MODEL_NOT_READY`**，前端可再次引导至设置 | P0 |
| 限流 | 按用户/IP/日预算 | P0 |
| 审计日志 | 会话创建、消息（可摘要）、索引任务、**用户模型配置变更**（不含明文 Key） | P0 |

### 5.6 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首 token 延迟可感知；流式优先 |
| 并发 | 小规模；峰值可参考 **数十并发会话** |
| 可用性 | 索引任务可重跑；LLM 失败降级话术 |

## 6. 数据与指标

| 指标 | 说明 |
|------|------|
| 有引用回答占比 | 政策/事实类期望高 |
| 索引失败率 | 解析与 embedding 稳定性 |
| 用户满意度 | 点赞踩 |
| 单会话 Token / 成本 | 成本控制 |
| 模型调用失败率 | 按提供方/模型维度观测，便于切换默认模型 |

### MVP 验收建议

- **同一知识库** 上传文档后，**能问出仅文档内有的信息**，且 **引用可对应原文**。
- **无相关内容** 时不胡编（验收用例需包含）。
- **新用户** 首次登录 **无法进入聊天页**，直至在 **模型设置页** 完成配置且 **readiness** 为 true；完成后 **可进入 `/chat`** 并完成对话与入库；**连通性测试** 通过。

## 9. 附录

| 名词 | 说明 |

## 10. 评审备忘（业务闭环与待收口项）

> 本节为 **自检结论**：便于迭代时补齐，不替代上文正式需求。

### 10.1 闭环自检

| 链路 | 是否闭环 | 说明 |
|------|----------|------|
| **配置 → 聊天** | 基本闭环 | A6b/A6c + readiness + `MODEL_NOT_READY` 已覆盖 |
| **上传 → 索引 → 问答** | 基本闭环 | A4/A5 + 门禁 + RAG；**删除文档/KB** 后的向量侧清理需在实现层收口（见下表） |
| **问 → 答 → 引用** | 闭环 | A3 + §5.3 |
| **反馈 → 优化** | **未闭环（MVP）** | 点赞/点踩为 **P1（A8）**；MVP 无显式「纠错工单」 |
| **成本/预算** | **部分缺口** | 目标表有「可管控」，**P0 未写用户侧预算与告警**（可 P1） |

### 10.2 待收口清单（建议在排期或详设中钉死）

| ID | 类型 | 说明 |
|----|------|------|
| G1 | 体验 | **无知识库** 时聊天：与 A2「无库可提示或禁止」一致，需 **统一空状态文案**（禁止上传 vs 仅提示） |
| G2 | 账号 | MVP 是否包含 **注册 / 找回密码**；若仅内测账号，需在 PRD 或发布说明中写明 |
| G3 | 数据 | **删除 KB/文档/Provider** 时：**Qdrant 点删除** 与 **MySQL 外键/级联** 策略 |
| G4 | 状态机 | 用户 **删除或禁用全部模型** 后 **readiness 变 false**，须再次 **禁止聊天**（与 A6c 一致） |
| G5 | 向量 | **更换默认 embedding**（A16）：与 **Qdrant collection 维度 / 重建任务** 的对应关系（详设已有方向，实现需任务表） |
| G6 | 合规 | 用户上传文件中的 **敏感信息**（PII）与 **可接受使用策略**（MVP 可仅提示「请勿上传机密」） |
